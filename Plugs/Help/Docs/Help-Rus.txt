
Плагин позволяет перейти в справку CHM, связанную с текущим типом документа, и открыть в ней выделенное слово, если оно есть в списке на вкладке "Указатель".

1. Файлы Plugs\Help.dll и папку Plugs\Help поместить в папки плагов
2. Сделать пункт меню или кнопку "Справка" Call("Help::Help") Icon("HH.exe", 0)
3. Выделите слово и нажмите кнопку

Параметры
0 - тихий режим, ни единого сообщения, полезен если нет желания разбираться с путями, а рука автоматом пытается вызвать справку для этого файла и вынуждает закрывать окно.
1 - по умолчанию, выдаёт только важные сообщения, "не найден путь к CHM", "Не найден ini-файл". Если неправильно скопирован плаг или неправильно указаны пути в ini-файле, или портабельная сборка перенесена и прямые пути не работают. Не обязательно указывать этот параметр, оставив пусто, это будет работать как "1".
2 - тест работы плага, выдаёт все возможные сообщения, чтобы найти проблему. Любая причина, по которой не открылась справка.

Пример с параметром:
"Тест плага Справка" Call("Help::Help", 2) Icon("HH.exe", 0)

Принцип работы плага:
При вызове функции используется выделенное слово и расширение файла взятые из открытого текущего документа. В ini-файле ищется секция с указанным расширением. Далее поиск окна с указанным в ini-файле заголовком, а если его нет, то запускается CHM, и снова поиска окна с заголовком. Далее ищется вкладки и активируется вкладка "Указатель", ищется поле для вставки слова и кнопка "Вывести/Показать", чтобы кликнуть на неё.

Поиск в интернете
"Поиск в интернете" Call("Help::HelpNet") Icon("%a\AkelFiles\icons\ico\google.ico")
Смотрите HelpNet.ini, чтобы наполнить меню пунктами. Секция [Default] будет отображаться всегда, а секции расширений, например [au3] будет отображаться только для этого типа файла. Для примера ссылки даны как для Googe, так и для Яндекса. Можно использовать любую ссылку, вместо текста используется %s, эта аббревиатура будет заменена на текст слова под курсором (с учётом регистра).
